Error UX rules:
• Never show a blank screen — always show an error state with a clear action
• Never lose user data — profile and household info is saved locally before generation starts
• Plain language only — no error codes, no technical messages (e.g. never "502 Bad Gateway")
• One retry CTA — not multiple options; "Try again" is the only action shown
• Auto-detect reconnection — app resumes without user action when connection returns
• Report a problem — if the issue persists after 2 retries, offer "Having trouble? Report a problem" as a secondary option
