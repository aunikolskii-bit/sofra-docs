Not "only cook what you already have." That would almost always fail.

HOW IT INTEGRATES
📥 Import → Auto-tag
Paste text, URL, or photo. System extracts recipe, estimates nutrition, detects dietary compliance tags. User reviews in 10 seconds.
🔍 Hard Rule Check
System scans ingredients against every household Hard Rule. If conflict found: clear warning + option to save as personal-only (not family plan).
⚖️ Enters the Planning Pool
Family recipes enter the meal selection engine alongside the curated library. Ranked by the same Soft Rules — time, budget, taste, variety.

Design notes
📸 Hero photo
Takes up the full width — food photography always looks better edge-to-edge. The ✨ Enhanced badge sits bottom-left so it's visible but doesn't block the food. Favourite star is bottom-right, always one tap away.
📝 Description block
A short personal description sits above the stats — this is where the emotional story lives. Orange left border signals "this is the human part." It's optional: auto-generated recipes use a neutral description; your own recipe gets your own words.
👨‍👩‍👧‍👦 Automatic portion sizing
The "Fits family" badge replaces a generic serving count. Men, women, and children have different energy needs — so the system silently assigns each household member a portion coefficient (adult male = 1.0, adult female ≈ 0.8, child ≈ 0.4–0.55). It then checks whether the recipe's stated servings actually cover your household and shows plain language: "just right", "a little extra", or "scale this up." The 420 kcal figure refers to a full adult portion — the kids' plates are proportionally smaller without the user needing to calculate anything.
⚖️ Ingredient unit toggle
Green = active unit. Tapping switches all quantities at once. No reload, no page jump — just the numbers change inline. Works for the whole ingredient list simultaneously.
🔢 Steps — progressive reveal
Steps 1–2 are fully visible. Step 3 fades out. The rest are locked behind "Start cooking" — this keeps the recipe page clean and gently nudges toward Cook Mode where the real experience lives.
▶ Primary action — Start cooking
The main CTA is in Warm Salmon Red — the appetite colour from our palette. It's the only red element on the page, so the eye goes there naturally after scanning the recipe.

🛒 Shop Mode — How It Works
Entry
Tap "Start Shopping" on the grocery list screen. The list re-sorts into store aisle order (Produce → Dairy → Meat → Bakery → Pantry) and big checkboxes appear.
During
Tap any item to check it off. Checked items dim and move to the bottom — remaining items always stay at the top. Progress bar shows how many you have left. Screen stays on automatically.
Exit
When all items are checked, the done screen appears. Tap "Back to my plan". Progress is saved — if you close the app mid-shop and return, everything you already checked stays checked.
Offline
Shop Mode works with no internet — most shops have poor signal. Checked-off state saves locally and syncs when back online. No connection needed for any part of the shopping experience.

What Was Found & Fixed
Issues in Prompt V1 — all resolved in Prompt V2
1
No recipe source defined
The prompt described the brain perfectly but never said where recipes come from. Fixed in V2: Curated hybrid model — 3,000 tested recipes + AI gap-filling for edge cases.
Critical
2
Phase 2 never defined
Phase 1 was clear. Phase 2 was referenced but never structured. Fixed in V2: Full Phase 2 definition — what triggers it, what it collects, how it's delivered (one question per session, not a form).
Critical
3
Fasting = one checkbox (severely underestimated)
Ramadan, Orthodox Christian fasts (180+ days/year), Jewish fast calendar, Hindu/Jain monthly fasts — each has completely different rules and dates. Fixed in V2: Calendar-aware, religion-specific fasting engine with annual updates.
Critical
4
Cultural flag required in Phase 0, not Phase 2
Saying "No restrictions in Phase 0" would cause the system to suggest pork to a Halal family on their very first visit. Fixed in V2: Cultural/religious flag captured at Phase 0 — before the first plan is generated.
High
5
Macro display contradiction
"Don't show percentages" + "Show Nutrition Overview (Calories + Macros)" are contradictory. Fixed in V2: Show calories + color-coded balance bar by default. Gram amounts available on tap. No percentages ever shown in the default view.
High
6
Hard Rules vs Soft Rules never distinguished
The prompt mixed safety rules (never override) with preference rules (adaptive) without naming the difference. Fixed in V2: Two-tier rule taxonomy explicitly defined — Hard Rules filter the pool, Soft Rules rank within it.
High
7
"Iron-focused" misclassified as a restriction
Iron-focused is a nutritional goal (boost iron-rich foods) — not a restriction (block certain foods). These require different logic. Fixed in V2: Moved to Goals category with ranking-boost behavior, not filtering behavior.
Medium
8
No variety / repetition prevention
Nothing stopped the system from suggesting the same "best" meal every day. Fixed in V2: Variety engine — no repeat within 14 days, no same main protein on consecutive days, cuisine rotation, seasonal scoring.
Medium
9
Pantry mode: no nutritional completeness signal
If a user has "rice, soy sauce, one egg" — the system would serve egg fried rice with no flag that it's nutritionally incomplete. Fixed in V2: One-line nudge when a key macro or nutrient group is missing from the pantry result.
Medium
10
Added Phase 0 (Ultra-minimal entry)
Not in V1 at all. Added in V2: A true zero-friction entry point — just family composition + cultural flag + cooking time. No goals, no restrictions, no explanations. Plan generated in under 60 seconds from first tap.

The service targets a global audience and Russian is a mandatory launch language. Added in V2: Full multilingual architecture — Russian + English at launch, auto-language detection, all UI strings, recipe content and AI output in the user's language, community recipes with translation-on-request, and Russian Orthodox fasting calendar as a first-class supported tradition.
